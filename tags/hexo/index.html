从零开始的个人博客建设

从零开始的个人博客建设

搭建阶段博客

写在前面

本教程属于完全零基础新手教学，适合所有计算机专业以及非专业人士参考学习建立自己的个人博客，并且全过程除了购买域名（类似于：你的名字.com）之外不会产生任何的费用，零基础读者不要对文章可能出现的些许代码产生恐惧，如果你使用的Windows系统那么将大大减少代码量同时也请你在参考文章教程是注意文章提醒的操作系统以进行相应操作，另外本文对于windows系统的教程也会相对细节许多。

为什么要建立自己的个人博客

笔者是计算机专业的学生，认为建立博客更多的是兴趣和爱好，锻炼自己个人能力同时记录整理自己的所学所感，笔者也非常鼓励非计算机专业的同学学习建立个人博客，因为在当今信息社会个人博客和个人网页相当于名片一样的存在，让你可以更好的在网络上展示自己。

需要准备的环境以及博客框架使用

本文的基本原理是通过Hexo框架搭建个人博客并远程部署到Github pages服务器上，如果读者需要可以购买域名，绑定以实现网站域名个性化（付费），购买网站会在之后提及，首先从准备环境到搭建仓库到SSH链接再到最后的个人博客大体框架搭建完成读者需要准备如下事项。

  1. Git
  2. 一个Github账号
  3. nodejs框架
  4. 本教程使用搭建博客的Hexo框架

工具下载和准备

1. Git的下载和安装：

Windows 系统下的安装: GIT官网下载地址 (下载完后一路next安装即可不需要额外操作)

Ubuntu（Liunx） 系统下载和安装简单很多只需要在控制台(Ctrl+alt+T)输入命令

```
sudo apt update
sudo apt install git
```

2. GitHub账户的注册

进入Github官网，根据提示完成注册（内地访问可能会有卡顿或者连接错误等现象本文章默认读者的网络条件良好）

3. nodejs框架和Npm的安装

Windows 系统下的安装: nodejs官网 点击进入官网下载适合自己系统的版本

Ubuntu（Liunx） 系统:（注意如果安装git时输入过了sudo apt update则无需再次输入）

```
sudo apt install nodejs npm
```

4. Hexo以及hexo-deployer-git的安装

Hexo是用于生成静态博客的，而hexo-deployer-git是用与之后将生成的静态博客上传到服务器中让你在互联网上可以有一个小窝
Windows 系统下的安装:
同时按下windows键+R键输入cmd后回车打开CMD命令窗口输入指令：

```
npm install hexo-cli -g
npm install hexo-deployer-git --save 
```

Ubuntu（Liunx） 系统:
打开端口输入如下命令：

```
 npm install -g hexo-cli
 npm install hexo-deployer-git --save
```

5. 安装完成检测

Windows 系统: 在CMD命令窗口输入软件版本检测命令：

 ```
 git --version
 node -v
 npm -v
 hexo -v
 ```

Ubuntu（Liunx） 系统：打开终端输入相同指令

搭建Github仓库

1. 进入Github官网登陆后点击左上角的绿色按钮”New”

![点击New建立新仓库](vx_images/276191318227043.png =453x)

2. 注意仓库的名字有强制要求，必须为”Owner“.github.io 其中Owner为注册时的id

这也是将来你博客的访问网站，https://owner.github.io， 你也可以通过 https://foevera0.github.io 来访问笔者的博客主页不过因为笔者购买了域名所以网址会被重新导向到笔者的域名上，但现在你还不能访问你的博客，因为你的仓库除了一个README文件外里空空是也。（笔者不能建立是因为笔者已经建立了一个博客仓库了）

![建立仓库](vx_images/222591518247209.png =492x)

在你的电脑上通过SSH连接刚刚创建的仓库（你已经完成了一半了！）

生成密钥并且全文复制

Windows 系统:
新建一个Blog文件夹用于存放你今后的博客文件或者在任何一处右键点击Git Bash Here进入Git界面

接着输入命令后按四次回车：

ssh-keygen -t rsa -C "你的Github账户邮箱"

前往生成密钥的文件夹打开id_rsa.pub文件（例如上文描述密钥生成目录为/c/Users/26091/.ssh/id_rsa.pub）
用记事本打开后全文复制（少一个字母都不行，多一个空格也不行）

Ubuntu（Liunx） 系统：你也需要新建一个blog空文件夹然后在文件夹内打开终端后输入

ssh-keygen -t rsa -C "你的Github账户邮箱"

按四次回车后前往生成目录打开id_rsa.pub文件

在GitHub官网绑定密钥

全文复制后打开浏览器回到Github主页，登陆后点击右上角头像菜单下的Settings进入设置界面进行SSH密钥绑定操作

在左侧菜单栏找到SSH and GPG keys进行绑定操作

![进入SSH and GPG keys选项进行绑定](vx_images/447071320226414.png =219x)

![点击New SSH key后粘贴刚才复制的密钥进行绑定操作](vx_images/536631420248854.png =520x)

![](vx_images/77451820243990.png =295x)

添加完成后回到Git Bash界面或Ubuntu(Linux)终端输入指令检测是否连接成功：

ssh -T git@github.com

![成功连接返回信息，ubuntu(linux)同理](vx_images/107671920237536.png =369x)

到这里你的电脑已经和GitHub pages成功建立了SSH连接！

初始化博客，简单两步你就可以浏览你的个人博客了！

hexo init命令

注意一定要在你准备放置博客文件的空文件夹blog下打开Git bash或Ubuntu(Linux)终端界面
输入初始化命令:

hexo init

如果返回错误信息中有:fatal:unable to access 'http://你的仓库地址'，代表与Github的连接不稳定（网络与github服务器连接不稳定）多试几次或者重启网络等方法可以解决。

hexo s命令， 你马上就可以看到你的个人博客了！

在刚才的终端输入命令：

hexo s

待指令执行结束后在你的浏览器地址栏内输入 https://localhost:4000 就可以看到自己的个人博客了！

到这里你已经完成了个人博客建设的一大半了，接下来是将你本地的博客远程部署到服务器上

在hexo初始化生成的文件中设置上传仓库

在生成博客的blog文件夹下找到新生成的_config.yml文件用笔记本或者其他编辑器打开后
下滑到最后一行将deploy: 下的代码修改成如下所示：

deploy: 
  type: git
  repository: git@github.com:你注册时的用户名/你注册时的用户名.github.io.git
  branch: main

注意最近Github的一次更新使得默认分支从master变成了main，同时所有的冒号( : )后都要有空格(不能紧挨着别的字符)
respository的地址由https改到ssh协议可以完美取代令牌验证繁琐问题。

hexo clean && hexo g && hexo d命令

以后也会经常用到的hexo命令三常客，可以结合在一起输入终端（一定要确保在你创建的blog文件夹下打开）：

hexo cl && hexo g && hexo d

笔者是用的Ubuntu(linux)系统，windows系统中终端返回相同信息

你的网站已经成功建立啦！！🎉

现在在浏览器里输入 https://你注册时的用户名/你注册时的用户名.github.io.git 你就可以看到你的个人博客啦！
如果你需要更改名字等信息可以再次打开_config.yml设置文件进行设置
